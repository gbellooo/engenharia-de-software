# UC001 - Registrar Incidente

**Aluno:** Gabriel Bello  
**RA:** 10416808  

## Ator Principal
Analista de Segurança

## Pré-Condições
- O analista está autenticado no sistema.
- O incidente foi identificado e possui informações mínimas para registro.

## Pós-Condições
- O incidente é cadastrado no sistema.
- Um protocolo de acompanhamento é gerado.
- O registro fica disponível para avaliação e tratamento posterior.

## Fluxo principal
1. O Analista de Segurança acessa a funcionalidade de registro de incidente.
2. O sistema apresenta a tela de cadastro do incidente.
3. O analista informa os dados principais do incidente.
4. O sistema envia os dados ao controlador responsável pelo caso de uso.
5. O controlador cria a entidade **Incidente** com as informações recebidas.
6. O controlador solicita a persistência do incidente no repositório.
7. O repositório confirma o armazenamento do incidente.
8. O sistema retorna ao analista a confirmação do cadastro com o protocolo gerado.

## Diagrama de Sequência

```plantuml
@startuml

@enduml
